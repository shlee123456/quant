## Codebase Patterns
- Project uses Korean commit messages (한글 커밋 메시지)
- Environment variables are documented in both .env.example and README.md
- Terminal logs are saved to .context/terminal/ with timestamps (excluded from git)
- KIS broker configuration uses KIS_* prefix for environment variables
- Dashboard helper functions go in dashboard/ directory (e.g., dashboard/kis_broker.py)
- Use `python3 -m pip install` for dependency installation
- Streamlit error messages should use st.error(), st.success() for user-friendly feedback
- Helper functions should return None on failure for graceful degradation
- KIS broker for US stocks: always use fetch_ticker(symbol, overseas=True, market='NASDAQ')
- KIS broker OHLCV for US stocks: use fetch_ohlcv(symbol, timeframe='1d', limit=N, overseas=True, market='NASDAQ')
- Streamlit metrics: use delta_color="normal" for green-up/red-down, "inverse" for opposite
- Color indicators: 🔴 for positive (red=up in Korean market), 🔵 for negative (blue=down)
- Plotly candlestick charts: use make_subplots(rows=2, cols=1, shared_xaxes=True) for price + volume layout
- Auto-refresh pattern for Streamlit: Store 'auto_refresh_enabled' and 'last_refresh_time' in session_state, use time.sleep(1) + st.rerun() for countdown timer
- Centralized error handling: Use dashboard/error_handler.py for consistent error messages across tabs
- Error messages should include: icon, title, description, expandable troubleshooting, and actionable solutions
- Dashboard tabs should check market_type and use_simulation before showing broker-specific features
- Always add translations for new UI elements in both English and Korean (dashboard/translations.py)
- Documentation in CLAUDE.md should include: overview, usage guide, code patterns, troubleshooting, and test checklist
- **TESTING CRITICAL PATTERNS** (2026-02-08 New Rules):
  - Position values MUST be 0 or 1 only (signal can be -1/0/1, but position is 0/1)
  - Strategy calculate_indicators() must handle empty DataFrames gracefully (check len(df) == 0)
  - Test data arrays in DataFrame creation must have equal lengths (validate before creating DataFrame)
  - KIS API tests should use @pytest.mark.slow and include time.sleep() between calls
  - Missing pytest fixtures should be defined in tests/conftest.py or local test file
  - KoreaInvestmentBroker requires user_id parameter (can default to account value)
